Отчёт по проекту, Tree Sort

1. Название алгоритма и краткая историческая справка 
Tree Sort – алгоритм сортировки, основанный на построении бинарного дерева поиска (Binary Search Tree, BST) из элементов массива и последующем симметричном (in‑order) обходе дерева для получения упорядоченной последовательности.

Концепция бинарного дерева поиска была известна с середины XX века, однако как самостоятельный метод сортировки Tree Sort оформился вместе с развитием структур данных в 1960‑х годах. В современных учебниках по алгоритмам Tree Sort обычно рассматривается как пример использования деревьев для сортировки и как база для построения более эффективных самобалансирующихся вариантов (например, AVL‑дерево или красно‑чёрное дерево).


2. Описание принципа работы и особенностей
Принцип работы Tree Sort состоит из двух основных этапов:
1) Построение бинарного дерева поиска (BST):
Создаётся пустое дерево.
Для каждого элемента входного массива выполняется вставка в BST по правилу: ключ меньше текущего узла → в левое поддерево, иначе → в правое.
Структура BST автоматически поддерживает свойство: для любого узла все ключи в левом поддереве меньше ключа узла, а в правом – больше (или равны, в зависимости от реализации).

2) Симметричный обход дерева (in‑order traversal):
Рекурсивно обходятся левое поддерево, текущий узел, правое поддерево.
При посещении узла его ключ добавляется в результирующий отсортированный массив.

Особенности алгоритма:

Требует дополнительной памяти для хранения узлов дерева (O(n)).
Время работы зависит от степени сбалансированности дерева.
Устойчивость зависит от реализации; в базовой версии (с равными ключами, идущими вправо) устойчивость не гарантируется.
При использовании самобалансирующихся деревьев (например, AVL) сложность становится гарантированно O(n log n).
 
3. Оценка сложности
Временная сложность в среднем: O(n log n) при случайном распределении входных данных (дерево получается примерно сбалансированным).
Временная сложность в худшем случае: O(n²) – возникает, если входные данные уже отсортированы или отсортированы в обратном порядке, что приводит к вырождению BST в линейный список.
Пространственная сложность: O(n) – для хранения n узлов дерева.

В данной работе реализована простая версия BST без балансировки, поэтому на практике будет наблюдаться близкая к теоретической O(n log n) на случайных данных, но при анализе худших сценариев может проявиться квадратичная зависимость.

6. Выводы
Реализованный алгоритм Tree Sort корректно сортирует массивы целых чисел.

Измерения времени и количества сравнений подтверждают теоретическую оценку средней временной сложности O(n log n) на случайных данных.

Графики показывают ожидаемый рост при увеличении размера входных данных, хотя присутствует некоторая дисперсия из‑за случайного характера данных и несбалансированности BST.

В худшем случае (например, на отсортированном массиве) время работы возрастает до O(n²), что является недостатком простой версии BST. При необходимости алгоритм можно улучшить, используя самобалансирующиеся деревья (AVL, красно‑чёрные), что гарантирует O(n log n) во всех случаях.

Дополнительная память O(n) для хранения дерева ограничивает применение Tree Sort на очень больших объёмах данных по сравнению с алгоритмами, работающими «на месте» (например, быстрой сортировкой).

Практические результаты согласуются с теоретическими ожиданиями, что подтверждает корректность реализации и методики измерений.
